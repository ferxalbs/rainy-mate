## 2023-10-27 - [React.memo in Streaming Chat Interfaces]
**Learning:** During streaming updates in React chat applications, components re-render extremely frequently (on every token). If child components (like trace views, plans, or artifact rows) are passed inline fallback arrays (e.g., `trace={message.trace || []}`) or unmemoized functions, `React.memo` on the child is defeated entirely due to referential inequality. This causes complex, unrelated UI sub-trees to fully tear down and re-render concurrently with streaming, drastically degrading frontend performance.
**Action:** Extract all fallback arrays to stable global constants (e.g., `const EMPTY_ARRAY: never[] = []`) outside the component body. Aggressively memoize complex UI elements within message lists using `React.memo()`, and strictly pair them with `React.useCallback()` and `React.useMemo()` in the parent for any functions or reference-type objects passed down.
